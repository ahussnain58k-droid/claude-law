# AI Governance Nuance

AI governance skills must track:

- role per system/use case
- tier per system/use case
- intended purpose
- actual use
- affected parties
- data categories
- autonomy level
- human oversight
- substantial modification
- reassessment triggers

Substantial modification examples:

- changed intended purpose
- new model
- new training data
- new user group
- new affected parties
- higher autonomy
- changed safety controls
- new deployment context
